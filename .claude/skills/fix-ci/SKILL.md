# CI失敗の修正

## 進め方
1. 失敗したCIのログを確認する。GitHub Actions の場合:
   `gh run list --limit 5` で直近の実行を確認し、
   `gh run view <run-id> --log-failed` で失敗ログを取得する。
   gh コマンドが使えない場合はユーザーにログの貼り付けを依頼する。

2. 失敗の原因を分類する。
   | 原因 | 対処 |
   |------|------|
   | テスト失敗 | 失敗しているテストを特定し、コードかテストのどちらが間違っているか判断する |
   | lint/format エラー | `ruff check` / `ruff format` を実行し、差分をコミットする |
   | 依存関係エラー | `uv lock` の実行忘れ、またはバージョン不整合を確認する |
   | 環境差異 | ローカルとCI環境の違い(Pythonバージョン、OS)を確認する |

3. 修正を実装する。修正は最小限に留める(CIを通すための変更だけ)。

4. ローカルで再現テストを実行してから push する。

## 注意
- CI失敗の修正ついでに、関係ない改善をしない(最小diff規律)。
- 原因が分からない場合は diagnosing-bugs スキルに切り替える。
