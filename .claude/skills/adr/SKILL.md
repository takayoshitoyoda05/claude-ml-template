# ADR記録

設計上の重要な決定を、後から理由を追える形で記録する。

## いつ書くか
- 複数の選択肢があり、どれか1つを選んだとき(トレードオフを伴う決定)
- 一度決めたら後から変更しづらい決定(データ形式、アーキテクチャ、外部依存の選定など)
- 些細な決定(変数名、フォーマットの好みなど)には書かない

## 進め方
1. 作業スコープ配下に docs/adr/ が無ければ作成する。
2. 既存のADRファイルを確認し、連番(0001, 0002, ...)の次の番号を決める。
3. templates/ADR.md.template の形式に沿って
   docs/adr/NNNN-<短いタイトル>.md を作成する。
4. 内容は「背景」「検討した選択肢」「決定」「理由」「トレードオフ」を埋める。
   簡潔に、しかし将来読んだ人が納得できる粒度で書く。

## 注意
- 決定を覆す場合は、古いADRを削除せず「このADRはNNNNにより置き換えられた」
  と追記し、新しいADRを追加する(履歴を残す)。
