# 文献レビュー(構造化された調査)

場当たり的な検索ではなく、スコープを固定してから調査する。
調査の再現性(なぜこの文献群を選んだか)を担保する。

## 進め方

### 1. スコープの固定
調査を始める前に、以下をユーザーと確認して literature/<テーマ>/scope.md に保存する。

```yaml
テーマ: <調査対象>
必須キーワード: [<キーワード1>, <キーワード2>]
除外条件: <対象外とする条件>
期間: <例: 2020年以降>
目的: <査読対応 / 関連研究セクション執筆 / 手法の比較検討>
```

### 2. 検索
- arXiv MCP が接続されていれば優先的に使う(検索・アブストラクト取得)
- 無ければ web_search で "site:arxiv.org <キーワード>" 等の検索を行う
- 検索クエリと取得結果は literature/<テーマ>/search-log.md に記録する
  (後で「なぜこの文献を見つけた/見逃したか」を追えるようにするため)

### 3. スクリーニング
各文献をタイトル+アブストラクトで判定し、以下に分類する。
- 採用: スコープに合致。詳細を読む
- 保留: 関連はあるが目的から外れる。理由を1行記録
- 除外: スコープ外。理由を1行記録

### 4. 抽出
採用文献ごとに以下を literature/<テーマ>/notes/<文献ID>.md に記録する。
- 書誌情報(著者、年、会議/誌名、URL)
- 1行要約
- 手法の要点(3行以内)
- 本研究との関係(何が同じで何が違うか)
- 引用する可能性のある主張とその根拠箇所

### 5. 統合
literature/<テーマ>/summary.md に統合レポートを書く。
- 文献同士の関係(系譜、対立、補完)
- 本研究の位置づけ(既存研究との差分)
- 見つかった研究ギャップ

## 重要なルール
- 存在しない文献をでっち上げない。確認できた文献のみを扱う
- アブストラクトしか読んでいない文献を「詳細を確認した」ように書かない。
  確認レベル(アブストのみ / 本文確認済み)を必ず記録する
- 本文に書かれていない結論を文献に帰属させない
