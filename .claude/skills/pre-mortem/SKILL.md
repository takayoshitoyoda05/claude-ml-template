# Pre-mortem 分析

動いているコードを見て、まだ起きていないバグの事後分析レポートを書く。
「今バグがあるか」ではなく「将来の編集で壊れやすい箇所はどこか」を探す。

## 進め方
1. 作業スコープ直下に CONTEXT.md があれば先に読む。
2. 対象のファイル・モジュールを指定される(無ければ作業スコープ全体)。
3. 各関数・クラスについて、以下の観点で脆さを分析する。

## 分析観点(1つずつ順に確認する。複数を同時に浅く見ない)
| 観点 | 具体的に見ること |
|------|------------------|
| 暗黙の仮定 | 入力の形状・型・値域について、コードが暗黙に仮定していることは何か |
| 状態依存 | グローバル変数・クラス変数・外部ファイルの状態に依存しているか |
| 順序依存 | 関数の呼び出し順序を変えると壊れるか |
| 数値の脆さ | 除算、log、sqrt 等でゼロやNaN/Infが入ったときに壊れるか |
| 次元・shape | テンソルの次元やshapeの仮定が、上流の変更で崩れる可能性はあるか |
| 外部依存 | ライブラリのバージョンアップで挙動が変わる箇所はあるか |

## 出力形式
各脆い箇所について、過去形で書く(「将来こう壊れた」という仮想の事後報告)。
