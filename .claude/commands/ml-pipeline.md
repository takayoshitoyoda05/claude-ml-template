# MLパイプライン修正フロー

3エージェントでモデル/評価コードの修正を実行します。

## 手順
1. planner に $ARGUMENTS を渡し、実装計画を作成させる
2. 計画をユーザーに提示し、承認されるまで進めない
3. generator に計画ファイルのパスを渡し、実装させる
4. evaluator に計画ファイルのパスと差分範囲を渡し、評価スクリプトを実行させて検証させる
5. evaluatorの判定:
   - PASS: 完了レポートを出力
   - NEEDS_REVISION: 指摘を整理して generator に差し戻す
   - FAIL(3回連続): planner からやり直すことを提案
6. 最大3イテレーションで収束しなければユーザーに判断を仰ぐ

$ARGUMENTS を要件として使用する。
